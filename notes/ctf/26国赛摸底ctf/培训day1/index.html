1. 模板目录改了

原来

改后

为什么改

原代码把这 3 个目录都当成 Jinja2 模板目录：

  • templates
  • userinfo
  • note

但 userinfo 和 note 这两个目录里的文件，是用户提交后动态写进去的。
这等于把“用户可控内容”放进模板引擎搜索路径里，风险非常大。

最典型的问题就是：

  • note/{userid}.html 会被当模板渲染
  • userinfo/{userid}.j2 也会被模板 import

这会产生 服务端模板注入 风险。

所以我保留了 templates 目录用来渲染：

  • index.html
  • readfail.html

但不再让 userinfo/ 和 note/ 参与模板解析。

为什么改

原来的读取方式，是把 note/{userid}.html 当 Jinja2 模板渲染。

这正是最危险的地方，因为这个 .html 文件本身是用户写入生成的。

我改成：

  • 直接读取 html 文件内容
  • 直接返回给浏览器
  • 不再经过模板引擎

这一步是整个 patch 的核心。

它保留了原本“访问 /note/{userid} 会展示笔记页面”的功能，但去掉了模板执行这一步。
