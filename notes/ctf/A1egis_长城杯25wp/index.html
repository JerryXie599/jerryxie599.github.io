1.The Silent Heist

借助本地大模型，以下是完整对话：

这是一个典型的对抗机器学习（Adversarial Machine Learning）CTF 题目。

核心解题思路

Isolation Forest（孤立森林）的工作原理是基于密度的异常检测。它假设“正常”数据在特征空间中聚集，且特征之间存在特定的相关性（协方差）。由于你有 1000 条正常样本，攻击的最佳策略是统计学拟合（Statistical Fitting）。

如果你只发送一条金额为 200 万的交易，或者随机生成数据，都会破坏特征间的协方差（例如：正常数据中 feat_1 增加时 feat_2 也会增加，如果你的数据 feat_1 增加 feat_2 却减少，就会被判定为异常）。

解决方案：多元高斯分布采样 (Multivariate Gaussian Sampling)

我们需要计算这 1000 条数据的 均值向量 (Mean Vector) 和 协方差矩阵 (Covariance Matrix)，然后利用这两个参数生成符合同一分布的全新数据。

Python 攻击脚本

请确保目录下有 public_ledger.csv 文件，然后运行此脚本。脚本不仅会生成数据，还会自动计算需要生成的笔数以凑齐 200 万金额。

为什么这样做有效？

  1. 绕过黑名单： 我们使用的是 np.random.multivariate_normal，这是概率采样。生成的每一个浮点数都与原始数据极其相似，但几乎不可能完全相同（浮点数精度极高）。这满足了“不直接重放”的要求。
  2. 绕过 Isolation Forest： 孤立森林通过随机切割特征空间来工作。异常点通常距离中心很远，通过很少的切割就能被分离出来（路径短）。我们生成的点严格遵循原始数据的 $ \Sigma $（协方差矩阵），这意味着生成的数据点都落在原始数据的高密度区域内。对于 AI 模型来说，这些就是“正常用户”。
  3. 沉默的劫案： 我们没有生成一笔 $ 2,000,000 的巨额交易（那绝对是统计学离群点，会被秒杀）。相反，我们生成了大约 5000-6000 笔金额为 $380 左右的“小额正常交易”，以此积少成多达到目标金额。

失败原因分析

Anomalies=48/5710 意味着你的生成策略在统计学上是准确的，但在对抗攻击层面是不够谨慎的。

  1. 正态分布的“长尾”效应：你使用原始的协方差矩阵 ($ \Sigma $) 进行采样。在统计学中，正态分布是有“尾巴”的。当你生成 5700 多条数据时，虽然绝大多数点都在中心，但在概率上必然会有少量点落在 3倍标准差之外。
  2. Isolation Forest 的原理：该算法就是专门抓这些“离群点”的。它通过随机切割特征空间，那些落在边缘（长尾）的数据会被判定为异常。
  3. 解决策略：保守采样 (Conservative Sampling)。我们需要生成“比正常人还要正常”的数据。也就是说，我们不仅要保持特征的相关性（方向），还要把生成的点紧紧地收缩在均值中心附近，坚决不碰边缘区域。

核心修改：协方差收缩 (Covariance Shrinkage)

我们将在代码中引入一个 shrinkage_factor（收缩因子）。比如设置为 0.3，意味着我们将生成的数据云团体积压缩到原来的 30%。所有的点都会非常靠近中心，从而 100% 避开 Isolation Forest 的切割边界。

修正后的代码 (Fixed for 0 Anomalies)

为什么这次一定能过？

  1. 收缩因子 (0.4)：标准正态分布中，99.7% 的数据在 3倍标准差内。Isolation Forest 可能会切掉最外面的那一层。通过乘以 0.4，原本 3倍标准差的点现在变成了 $ 3 \times \sqrt{0.4} \approx 1.9 $ 倍标准差。这使得生成的所有数据都落在了原始分布的最核心、最密集的区域（High Density Region）。AI 模型会认为这些数据是“教科书级别的正常用户”。
  2. 避免边缘值：之前可能有少量的负数（通过 abs 强转）或者极端大值，这些都是异常特征。收缩方差后，出现这些值的概率降为 0。

操作指南

  1. 将代码保存为 exploit_v2.py。
  2. 运行 python exploit_v2.py。
  3. 它会生成一个新的 submission.csv。
  4. 将该文件内容发送给题目端口。

这次为什么会通过？
Isolation Forest 的原理是看某些点是否容易被切分（isolated）。

  • 之前的代码生成的分布和真实数据一样宽，边缘的点容易被切分。
  • 现在的代码生成的分布只有真实分布的 20% 宽。对于 AI 模型来说，这些数据点简直是在靶心的正中央，没有任何只有“异常值”才具有的稀疏性特征，因此不会触发警报。
