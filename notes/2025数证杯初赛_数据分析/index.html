89、通过对检材“01-lott.sql”文件进行分析，统计庄家”188”在2021-05-10当日的电子投注总笔数（答案格式：1）（ ）

  1. 目标：统计庄家 "188" 在 2021-05-10 当日的电子投注总笔数。
  2. 数据来源：
    • 数据库：lott
    • 表名：通过查询得知为 t_trade_betrecord。
  3. 关键字段：
    • 庄家 ID 字段：Keeper（代表庄家编号）。
    • 投注时间字段：CreateTime（格式为 YYYY-MM-DD HH:MM:SS）。
  4. 过滤条件：
    • Keeper = '188'：筛选出庄家为 "188" 的记录。
    • 时间范围：CreateTime >= '2021-05-10 00:00:00' AND CreateTime <= '2021-05-10 23:59:59'，确保只统计当天的数据。
  5. 结果要求：
    • 使用 COUNT(*) 统计符合条件的记录总数。
  • SELECT - 查询关键字，用于从数据库中检索数据
  • COUNT(*) - 聚合函数，用于计算行数（记录总数），* 表示计算所有行
  • AS total_bets - 给查询结果起一个别名（列名），将计数结果命名为 total_bets

解释：

  • COUNT(*)：计算满足 WHERE 条件的所有行数。
  • Keeper = '188'：筛选庄家编号为 "188" 的投注记录。
  • CreateTime 范围限制在 2021-05-10 整天的时间范围内。

此查询直接从数据库中提取并统计了指定条件下的投注笔数，最终返回的结果即为所需答案。

    • 投注日期为 2021-05-10：通过 CreateTime 字段进行筛选

  3. 统计逻辑：
    • 按照投注内容 (Content 字段) 进行分组
    • 统计每个投注内容的出现次数
    • 按出现次数降序排列
    • 取第一条记录作为频率最高的内容

92、通过对检材“01-lott.sql”文件进行分析，关联t_trade_betrecord与t_lottery_jnd表，分析庄家”188”在2021-05-10投注”jnd28”时：当开奖结果为”大”时，玩家投注包含”小”的笔数占比（使用双位数格式，答案格式：11.11%）（ ）

我们需要分析庄家”188”在2021-05-10投注”jnd28”时，当开奖结果为”大”时玩家投注包含”小”的笔数占比。具体步骤如下：

  1. 表关联：
    • 关联 t_trade_betrecord 和 t_lottery_jnd 表，通过 Issue 字段进行连接
  2. 过滤条件：
    • 庄家为 “188”：通过 Keeper 字段筛选
    • 彩票类型为 “jnd28”：通过 LotteryType 字筛选
    • 投注日期为 2021-05-10：通过 CreateTime 字段筛选
    • 开奖结果为”大”：通过 t_lottery_jnd.R_B_M 字段筛选
  3. 计算逻辑：
    • 统计开奖结果为”大”时的总投注笔数
    • 统计其中投注内容包含”小”的笔数
    • 计算占比并格式化为双位数百分比

实际执行的 SQL 查询代码如下：

CASE WHEN b.Content LIKE '%小%' THEN 1 ELSE 0 END

R_B_M大小

查询说明：

  • JOIN lott.t_lottery_jnd j ON b.Issue = j.Issue：关联两个表，通过期号连接
  • b.Keeper = '188'：筛选庄家为”188”的记录
  • b.LotteryType = 'jnd28'：筛选彩票类型为”jnd28”的记录
  • DATE(b.CreateTime) = '2021-05-10'：筛选2021年5月10日的记录
  • j.R_B_M = '大'：筛选开奖结果为”大”的记录
  • CASE WHEN b.Content LIKE '%小%' THEN 1 ELSE 0 END：判断投注内容是否包含”小”
  • SUM(...) * 100.0 / COUNT(*)：计算包含”小”的投注笔数占总笔数的百分比
  • FORMAT(..., 2)：格式化为保留两位小数的数值
  • CONCAT(..., '%')：添加百分号

根据验证查询，开奖结果为”大”时总投注笔数为1161笔，其中包含”小”的投注笔数为521笔，占比为 44.88%。

答案：44.88%

93、通过对检材“02-crime_records.sql”分析，统计相邻两次作案时间间隔在1天之内的城市和该城市两次作案时间间隔在1天之内的案件总数量，找出案件总数最多的城市名。（答案格式：按照实际情况填写）（ ）

我们需要找出相邻两次作案时间间隔在1天之内的城市，并统计每个城市中这样的案件对数量，最后找出案件对总数最多的城市。具体步骤如下：

  1. 数据预处理：
    • 使用CTE（公用表表达式）ordered_crimes对每个城市的犯罪记录按时间排序
    • 使用STR_TO_DATE函数将字符串格式的时间转换为日期时间格式
    • 使用ROW_NUMBER()为每个城市的记录分配序号
  2. 关联相邻记录：
    • 使用CTEcrime_pairs将同一城市中相邻的两条记录关联起来
    • 计算相邻案件之间的时间差（小时）
  3. 筛选和统计：
    • 筛选时间差在24小时（1天）之内的案件对
    • 按城市分组统计符合条件的案件对数量
    • 按数量降序排列，找出最多的那个城市

实际执行的 SQL 查询代码如下：

查询说明：

  • STR_TO_DATE(crime_time, '%Y/%m/%d %H:%i')：将字符串格式的时间转换为日期时间格式
  • ROW_NUMBER() OVER (PARTITION BY city ORDER BY ...)：为每个城市的记录按时间顺序编号
  • JOIN ordered_crimes c2 ON c1.city = c2.city AND c1.rn = c2.rn - 1：关联同一城市中相邻的两条记录
  • TIMESTAMPDIFF(HOUR, c1.crime_datetime, c2.crime_datetime)：计算两条记录之间的时间差（小时）
  • ABS(hours_diff) <= 24：筛选时间差在24小时（1天）之内的记录对
  • GROUP BY city：按城市分组
  • COUNT(*)：统计每个城市的符合条件的记录对数量
  • ORDER BY pair_count DESC：按数量降序排列
  • LIMIT 1：只取数量最多的第一个城市

根据查询结果，福州是相邻两次作案时间间隔在1天之内的案件对数量最多的城市，共有2076对。

答案：福州

94、通过对检材“02-crime_records.sql”分析，根据案件的损失金额和伤情等级，将案件分为 “轻微案件”“一般案件”“重大案件”“其他”四类（分类规则如下），并统计 2023 年各类型案件的数量。轻微案件：损失金额≤10000 元且无人员受伤（injury_level 为空或未提及）；一般案件：损失金额 10001-50000 元，或有轻微伤；重大案件：损失金额 > 50000 元，或有轻伤或有重伤；其他：非上述情况。（按照案件数量的降序输出答案，答案格式为：40/30/20/10）（ ）好的，我来详细解释查询逻辑和查询语句：

查询逻辑

分类规则理解

根据题目要求，案件分为四类，需要按优先级顺序判断：

  1. 重大案件（优先级最高）：损失金额 > 50000 元，或伤情等级为”轻伤”或”重伤”
  2. 一般案件（优先级第二）：损失金额 10001-50000 元，或伤情等级为”轻微伤”
  3. 轻微案件（优先级第三）：损失金额 ≤ 10000 元，且无人员受伤（injury_level 为空或未提及）
  4. 其他：不符合以上三类的情况

关键点

  • CASE WHEN 语句从上到下依次判断，遇到第一个满足的条件即停止
  • 因此需要按优先级从高到低排列条件
  • “有重伤”包括 injury_level 为 “轻伤” 或 “重伤”
  • “无人员受伤”指 injury_level 为 NULL、空字符串，或不包含具体伤情描述
